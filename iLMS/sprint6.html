Sprint 6

i-LMS project

Sprint Plan

In this sprint, the plan was to complete working on the user-interface of coding questions. Also to ensure that on refreshing the browser, the student does not lose the submitted data. We also planned to add test configurations which will allow the assessment create to decide whether the student can navigate to previous questions or not.

We also decided to keep the evaluation and results of fill up questions on hold for this sprint.

User Stories

In this sprint, we continued with the user-interface design and page refresh user-stories and started with a new user-story for test configurations.

  1. As a student, if I refresh the page, I should remain on same page and not lose my progress (Continued from previous sprint).
  2. As a student, I should have a user-interface for attempting the assessment which is comfortable to use (Continued from previous sprint).
  3. As an assessment creator, I should be able to configure the navigation in the test according to my requirements.

My Role

Since in the previous sprint I was working on the user-interface, so I decided to continue with the same user-story.

In this sprint, I decided to change the button for collapsing and expanding. I completely expanded the column to almost full width of the screen and replaced the other column with a button.

In problem statement, there is sample input and output. Also it is possible that the assessment creater wants to give some pre-written code to the students. For that I changed the format of question in JSON file.

After making changes in the JSON file, the final design for the coding question was like this.

After that I worked on adding backward navigation, i.e, giving an option to allow the student to move back to previous questions. For this I added a boolean to the JSON which will be check whether studenets can go back or not.

Sprint Outcome

In this sprint, we were able to complete the user-interface for the assessment system. Also functionality for not losing the data on refreshing was implemented.

For backward navigation, movement to previous question was implemented along with deactivation of button on first question. However, we need to work on displaying the answers for attempted questions on going back to that question.
