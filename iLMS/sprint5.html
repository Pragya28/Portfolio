Sprint 5

i-LMS project

Sprint Plan

In this sprint, the plan was to complete the user-stories which were incomplete in the previous sprint. Also to store the answers entered by a student and make some changes in the user-interface along with making it responsive.

User Stories

In this sprint, we were continuing previous user-stories and had new stories for user-interface and data storage.

  1. As a student, I should be able to know my score and detailed result after completion of the assessment (Continued from previous sprint).
  2. As a student, if I refresh the page, I should remain on same page and not lose my progress (Continued from previous sprint).
  3. As a student, I should have a user-interface for attempting the assessment which is comfortable to use.
  4. As a student, the answers I entered should get stored somewhere.

My Role

I concentrated on creating new user-interface for coding question in a way that it does not looks congested and a student can easily read the problem statment and has enough space to write code.

I initally came up with a three column layout - first for problem statment, second for code editor and third for output window, in which click on a button can expand or collapse a column.

This design was dropped because if all the columns are open then we are getting very less column width. Also the buttons also caused some confusions regarding which column is open or close. So solve that problem, I increased the size of the card in which question needs to be displayed and made a two column layout - first for problem statment and second having code editor on top and output window at bottom.

Because of changing the dimensions of the card in which questions were being displayed, the look of fill up questions also got changed a bit.

Along with design for desktop view, I also ensure that the design was responsive and can work on mobiles.

Sprint Outcome

In this sprint, we were able to ensure that anytime the assessment is refreshed it remains on same page. We also changed the user-interface of the assessment in terms of spacing and placement of components. Lastly, we were able to store the answers written by a student for fill-ups questions in session storage.

Few things from this sprint which remain incomplete is giving the result of the assessment (fill-ups only) after test is finished. Some changes in the coding questions for resizing of the two columns. We also need to start working on coding assessment for the next sprint.
