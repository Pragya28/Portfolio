Sprint 7

i-LMS project

Sprint Plan

In this sprint, the plan was to work with the session storage to store the data for coding questions, time taken by student to complete the test and showing already written answers on back navigation.

We also worked on completing the result for the fill-ups.

Along side, we also fixed some issues which we found during our work.

User Stories

In this sprint, we continued with the result user-story from sprint 5 which was put on hold for sprint 6. We also started working on new user-stories for storage and time.

  1. As a student, I should be able to know my score and detailed result after completion of the assessment (Continued from sprint 5).
  2. In results, as a student, I should be able to see how much time did I took to complete the assessment.
  3. As a student, if I go back to a question, I should be able to see and edit the answers of already answered questions.

My Role

In fill-ups we were facing an issue of not being able to give proper indentation where require in the code. To solve this problem we made some changes in the JSON file structure and added an not-editable code editor there, so that we can have proper indentation and syntax highlighting.

Sprint Outcome

With this sprint, the assessment system is almost ready and the only remaining part is execution and evaluation of coding questions.
